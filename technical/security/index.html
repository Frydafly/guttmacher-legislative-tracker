    Airtable Security and Data Management Overview¶

    Current Implementation¶

    - Teams Plan Account

    - 5 Editor Seats

    - Interface sharing via secure links

    - Role-based access control

    Data Security Features¶

    Infrastructure Security¶

    - SOC 2 Type II certified

    - ISO 27001 certified

    - 256-bit TLS encryption in transit

    - AES-256 encryption at rest

    - AWS infrastructure (us-east-1 region)

    - Regular third-party security audits

    Access Control¶

    1. Base-Level Security

    - Granular permissions per base

    - Domain-restricted sharing

    - Audit logs of all access

    - Session management and timeout controls

    2. Team Management

    - Role-based access (Owner, Creator, Editor, Commenter, Read-only)

    - IP address restriction capability

    - Multi-factor authentication support

    - Single sign-on (SSO) available

    - Centralized user management

    3. Interface Security

    - Restricted sharing via secure links

    - Configurable permissions per interface

    - Access logging for interface usage

    - Ability to revoke access at any time

    Data Protection¶

    1. Backup and Recovery

    - Continuous data backup

    - Point-in-time recovery (up to 30 days)

    - Deletion recovery within retention period

    - Data export capabilities for local backup

    2. Version Control

    - Field-level change history

    - Record revision tracking

    - Ability to restore previous versions

    - Audit trail of all modifications

    3. Data Retention

    - Configurable retention policies

    - Automatic archival options

    - Data export for long-term storage

    - Compliance with retention requirements
